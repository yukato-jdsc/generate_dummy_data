# Full Output Size Design

## Status

この設計は 2026-04-22 時点で廃止。現行仕様では出力サイズ目標を持たず、全カラム非空欄を優先する。

## 背景

`docs/format.md` に各 CSV の想定データ量が追記された。
現在の CSV 生成は `--full` 時の件数は合わせているが、出力サイズはその想定から大きくずれている。

確認済みの現状サイズは次のとおり。

- `m_campaign_all.csv`: 約 198KB
- `m_agency_all.csv`: 約 2.14GB
- `m_agency_diff.csv`: 約 98KB
- `m_product_all.csv`: 約 93.7MB

一方、`docs/format.md` の想定は次のとおり。

- `m_campaign_all.csv`: 400KB
- `m_agency_all.csv`: 300MB
- `m_agency_diff.csv`: 80KB
- `m_product_all.csv`: 219.76MB

つまり、`agency` 系は出力が重すぎ、`campaign` と `product` は軽すぎる。件数を維持したまま、`--full` 時の平均行長を CSV ごとに調整する必要がある。

## 目的

- `--full` 実行時の出力サイズを `docs/format.md` の想定に近づける
- 通常実行時の軽量な使い勝手は維持する
- 既存の件数、列構造、差分 CSV の部分集合関係は壊さない

## 採用方針

`--full` 時だけ、CSV ごとのサイズ調整プロファイルを適用する。

- `campaign`: 可変長テキスト列を少し厚めに埋める
- `agency`: 可変長テキスト列の埋め率と文字数を大きく抑える
- `product`: 可変長テキスト列を中程度に厚めに埋める
- `agency_diff`: `agency_all` からの抽出を維持し、独自調整は行わない

この方式により、現在のずれ方が CSV ごとに逆方向である問題に素直に対応できる。生成後の再試行や逐次最適化は導入しない。

## 変更対象

### `csv_generator/config.py`

- `--full` 時のサイズ調整プロファイルを定義する
- プロファイルには少なくとも次を持たせる
  - 対象 CSV
  - 可変長列のカテゴリ
  - 各カテゴリの文字数帯
  - 空欄率または簡略化率

### `csv_generator/generators.py`

- `--full` かどうかに応じてサイズ調整プロファイルを参照する
- 固定長に近いコード列、日付列、数値列の生成は変更しない
- 名称、説明、英語名、分類名などの可変長列だけを調整対象にする
- `agency_all` の出力密度を下げる一方で、`campaign` と `product` は適度に情報量を増やす
- `agency_diff` は従来どおり `agency_all` のリザーバサンプリング結果を書き出す

### `tests/test_generate_csv.py`

- 既存の件数確認と差分部分集合確認を維持する
- `--full` 実行時のサイズ検証を追加する
- サイズ検証は厳密一致ではなく、想定サイズに対して実用的な許容幅に収まることを確認する

## アーキテクチャ

件数制御とサイズ制御を分離する。

- 件数は既存の `DEFAULT_COUNTS` / `FULL_COUNTS` をそのまま使う
- サイズ制御は `--full` 時にだけ有効化する
- 行生成時に、対象 CSV に対応するプロファイルをもとに可変長テキストの密度を調整する

この構成なら、通常実行は引き続き軽量なまま維持でき、`--full` のみ要件に寄せられる。

## コンポーネント設計

可変長列を直接すべて個別調整するのではなく、列の性質ごとにカテゴリ分けして扱う。

想定カテゴリの例:

- 名称系: 正式名称、略称、カナ名称、英語名称
- 説明系: `description` のような自由文
- 分類名系: 区分正式名称、カテゴリ名称、部門名
- 住所系: 付帯住所などの補足文字列

各 CSV は、これらのカテゴリに対して別々の密度設定を持つ。

- `campaign` は名称系と説明系を厚めにする
- `agency` は名称系と分類名系を短めまたは空欄寄りにする
- `product` は名称系と分類名系を厚めにする

これにより、どの列がサイズ増減に効くかをコード上で説明しやすくなる。

## データフロー

1. `docs/format.md` を読み込んで列定義を取得する
2. CLI で `--full` を判定する
3. `--full` の場合だけ対象 CSV のサイズ調整プロファイルを有効化する
4. 行生成時に可変長列の生成結果へプロファイルを適用する
5. CSV を書き出す
6. `agency_diff` は `agency_all` の出力行から従来どおり抽出する

## エラーハンドリング

- 未知の target 指定、仕様読込失敗などの既存エラー処理は変更しない
- サイズ調整はベストエフォートとし、想定サイズへの厳密一致を保証する機構は追加しない
- 目標サイズとの差分が残っても異常終了にはしない

## テスト方針

次を自動テストで担保する。

- `--full` でも既存の件数が変わらないこと
- `agency_diff` が `agency_all` の部分集合であること
- `--full` 時の出力サイズが `docs/format.md` の想定に対して大きく外れないこと

サイズ検証は厳密一致ではなく許容幅で判定する。理由は、CSV の BOM、改行、引用符付与、ランダム値のばらつきで数値が多少前後するためである。

## 非対象

- 通常実行時のデータサイズ調整
- `docs/format.md` の件数やサイズ定義の変更
- 出力 CSV の文字コード変更
- 生成後にファイルサイズを見て再生成するフィードバック制御
